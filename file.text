📘 4. Relational Database Concepts ⭐⭐ (MOST IMPORTANT)
This is one of the most important topics for MCQs and practical exams. Almost every paper includes questions from here.

🔹 1. What is Relational Model?
The Relational Model stores data in the form of tables (relations).

👉 Introduced by E.F. Codd

🔹 Structure of a Table

https://www.datacadamia.com/_media/data/modeling/relational_data_model.jpg?fetcher=raw&tseed=1725012530

https://miro.medium.com/1%2Awr_PNTP9fQHxXeMydaSfnQ.jpeg

https://www.cs.sjsu.edu/~pearce/modules/lectures/oop/persistence/Databases_files/image006.jpg
4
Key Terms:
Relation → Table

Tuple → Row

Attribute → Column

Domain → Set of valid values

Schema → Structure of table

Instance → Actual data

🔹 2. Keys in Relational Model ⭐
✅ Types of Keys:
🔸 Primary Key
Uniquely identifies each record

Cannot be NULL

🔸 Foreign Key
Links two tables

Refers to primary key of another table

🔸 Candidate Key
Possible primary keys

🔸 Composite Key
Combination of multiple attributes

🔸 Super Key
Any set that uniquely identifies rows

🔹 3. Relationships Between Tables
Types:
One-to-One (1:1)

One-to-Many (1:N) ⭐

Many-to-Many (M:N)

Example:
One student → Many courses (1:N)

Many students ↔ Many courses (M:N)

🔹 4. Integrity Constraints ⭐⭐
✅ Types:
🔸 Entity Integrity
Primary key cannot be NULL

🔸 Referential Integrity
Foreign key must match primary key

🔸 Domain Integrity
Valid data types & values

🔹 5. Relational Schema Example
Student Table:
Student(ID, Name, Age)
Course Table:
Course(CourseID, CourseName)
Enrollment Table:
Enrollment(StudentID, CourseID)
👉 This resolves Many-to-Many relationship

🔹 6. Degree & Cardinality
Degree → Number of columns

Cardinality → Number of rows

🔹 7. Advantages of Relational Model
Simple structure

Easy querying (SQL)

Data consistency

Flexible
